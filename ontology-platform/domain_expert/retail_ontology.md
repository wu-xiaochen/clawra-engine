# 零售领域本体设计

## 概述

本文档定义零售领域的核心本体模型，涵盖商品管理、门店运营、客户关系、订单处理、库存管理、促销活动等核心业务领域，用于指导零售知识库构建和智能化应用。

---

## 一、概念定义 (Concepts)

### 1.1 核心实体

| 概念 | 英文 | 定义 | 关键属性 |
|------|------|------|----------|
| 商品 | Product | 零售企业销售的有形或无形产品 | 商品编码、名称、规格、品类、价格、成本 |
| 品牌 | Brand | 商品的品牌标识 | 品牌编码、名称、级别、授权状态 |
| 品类 | Category | 商品的分类体系 | 品类编码、名称、层级、关联品类 |
| 供应商 | Supplier | 提供商品的企业主体 | 供应商编码、名称、资质、账期 |
| 门店 | Store | 零售交易的线下场所 | 门店编码、名称、地址、面积、业态 |
| 客户 | Customer | 购买商品的个人或企业 | 客户ID、姓名、等级、联系方式 |
| 会员 | Member | 注册并享受会员权益的客户 | 会员ID、等级、积分、注册时间 |
| 订单 | Order | 客户购买商品的交易记录 | 订单编号、客户ID、金额、状态、时间 |
| 库存 | Inventory | 商品在门店或仓库的实际数量 | 商品ID、门店/仓库ID、数量、批次 |
| 促销 | Promotion | 促进销售的营销活动 | 促销ID、名称、活动类型、规则、周期 |
| 收银 | Checkout | 门店的商品结算交易 | 收银ID、门店ID、操作员、金额、支付方式 |
| 退货 | Return | 客户退回商品的操作 | 退货单号、原订单ID、退款金额、原因 |

## 二、规律 (Laws)

### 2.1 零售公式

#### 2.1.1 销售额公式

```
销售额 = 客流量 × 转化率 × 客单价 × 复购率
```

**分解说明**:
- 客流量 (Traffic): 进店顾客数量
- 转化率 (Conversion Rate): 成交顾客/进店顾客
- 客单价 (Average Transaction Value): 每笔订单平均金额
- 复购率 (Repurchase Rate): 重复购买顾客占比

#### 2.1.2 毛利率公式

```
毛利率 = (销售收入 - 销售成本) / 销售收入 × 100%
毛利润 = 销售收入 × 毛利率
```

#### 2.1.3 坪效公式

```
坪效 = 销售额 / 经营面积 (元/㎡)
```

**行业基准**:
- 便利店: 2-4万元/㎡/年
- 超市: 1.5-3万元/㎡/年
- 百货: 2-5万元/㎡/年

#### 2.1.4 人效公式

```
人效 = 销售额 / 员工人数 (万元/人/年)
```

### 2.2 库存周转规律

#### 2.2.1 库存周转率

```
库存周转率 = 期间销售成本 / 平均库存成本
周转天数 = 365 / 周转率
```

**行业基准**:
- 便利店: 10-15次/年 (24-36天)
- 超市: 8-12次/年 (30-45天)
- 服装: 4-8次/年 (45-90天)
- 数码电器: 5-8次/年 (45-73天)

#### 2.2.2 库存周转天数警戒线

| 品类 | 正常范围 | 预警 | 积压风险 |
|------|----------|------|----------|
| 生鲜 | 1-3天 | >5天 | >7天 |
| 食品 | 15-30天 | >45天 | >60天 |
| 日用品 | 30-45天 | >60天 | >90天 |
| 服装 | 60-90天 | >120天 | >180天 |

### 2.3 客户价值规律

#### 2.3.1 RFM模型

```
R (Recency): 最近一次消费时间 - 越近越好
F (Frequency): 消费频率 - 越高越好
M (Monetary): 消费金额 - 越大越好
```

**客户分层**:
- 高价值客户: R/F/M三项指标均高
- 成长客户: F/M高但R较低
- 流失风险客户: R很低
- 低价值客户: F/M均低

#### 2.3.2 客户生命周期价值 (CLV)

```
CLV = (平均客单价 × 购买频率 × 平均客户生命周期) × 毛利率
```

### 2.4 促销效果规律

#### 2.4.1 促销敏感度

```
促销提升系数 = 促销期间销售额 / 平时销售额
```

**常见促销效果**:
- 折扣 (10% off): 提升 20-40%
- 买赠: 提升 15-30%
- 满减: 提升 25-50%
- 会员价: 提升 10-20%
- 组合促销: 提升 15-35%

#### 2.4.2 促销毛利率影响

```
促销后毛利率 = 促销前毛利率 - 折扣损失率
折扣损失率 = (原价 - 促销价) / 原价
```

### 2.5 损耗率规律

```
损耗率 = 损耗金额 / 销售额 × 100%
```

**行业基准**:
- 便利店: 1-2%
- 超市: 1.5-2.5%
- 生鲜: 3-8%
- 服装: 2-4%

**损耗原因分布**:
- 过期报损: 30-40%
- 盗窃: 20-30%
- 破损: 15-20%
- 盘点误差: 10-15%

## 三、业务规则 (Rules)

### 3.1 商品管理规则

#### 3.1.1 商品准入规则

| 条件 | 要求 |
|------|------|
| 资质要求 | 具备营业执照、生产许可证、质检报告 |
| 品牌授权 | 需提供品牌方授权书或正规进货渠道证明 |
| 质量标准 | 符合国家相关质量标准要求 |
| 价格竞争力 | 建议零售价不高于同类商品均价10% |
| 供应保障 | 供应商需保证持续供货能力 |

#### 3.1.2 商品定价规则

```
建议零售价 = 成本 × (1 + 加价率)
```

**品类加价率参考**:
- 生鲜: 15-25%
- 食品: 20-35%
- 日用品: 25-40%
- 服装: 50-100%
- 数码电器: 8-15%

#### 3.1.3 商品上下架规则

| 场景 | 规则 |
|------|------|
| 上架 | 完成商品信息维护、定价、库存初始化 |
| 下架 | 滞销超过90天、质量问题、供应商断供 |
| 停售 | 过期、召回、违规商品立即下架 |

### 3.2 库存管理规则

#### 3.2.1 补货规则

```
补货量 = (目标库存 - 当前库存) × 补货系数
```

**补货触发条件**:
- 库存 ≤ 安全库存
- 预计可售天数 < 最低可售天数

**安全库存计算**:
```
安全库存 = 平均日销量 × 补货周期 × 安全系数
```

#### 3.2.2 库存预警规则

| 预警级别 | 条件 | 动作 |
|----------|------|------|
| 绿色 | 库存 > 安全库存×1.5 | 正常销售 |
| 黄色 | 库存 ≤ 安全库存 | 触发补货 |
| 橙色 | 库存 ≤ 安全库存×0.5 | 紧急补货 |
| 红色 | 库存 = 0 | 缺货，暂停销售 |

#### 3.2.3 盘点规则

| 门店类型 | 盘点频率 | 盘点方式 |
|----------|----------|----------|
| 便利店 | 每月1次 | 全面盘点 |
| 超市 | 每周1次 | 分区盘点 |
| 仓储店 | 每季度1次 | 全面盘点 |

### 3.3 促销管理规则

#### 3.3.1 促销活动类型

| 促销类型 | 说明 | 规则示例 |
|----------|------|----------|
| 折扣促销 | 直接降价 | 8折、7.5折 |
| 满减促销 | 达到金额减扣 | 满100减10 |
| 买赠促销 | 购买赠送 | 买二送一 |
| 组合促销 | 关联商品组合 | 套餐价 |
| 会员专享 | 会员专属价格 | 会员价9折 |
| 新品推广 | 新品优惠 | 首单半价 |

#### 3.3.2 促销审批规则

| 促销预算 | 审批人 |
|----------|--------|
| < ¥10,000 | 门店店长 |
| ¥10,000 - ¥50,000 | 区域经理 |
| ¥50,000 - ¥200,000 | 营销总监 |
| > ¥200,000 | 总经理 |

#### 3.3.3 促销时间规则

- 单次促销周期: 最长30天
- 同一商品促销间隔: 至少7天
- 节日促销提前申请: 提前15天

### 3.4 客户管理规则

#### 3.4.1 会员注册规则

| 注册方式 | 所需信息 | 权益 |
|----------|----------|------|
| 手机号注册 | 手机号、验证码 | 积分累计 |
| 实名注册 | 姓名、手机号、身份证 | 积分+优惠 |
| 企业客户 | 企业信息、联系人 | 对公结算 |

#### 3.4.2 会员等级规则

| 等级 | 升级条件 (年消费) | 折扣 | 积分倍率 |
|------|-------------------|------|----------|
| 普通 | < ¥1,000 | 无 | 1倍 |
| 银卡 | ¥1,000 - ¥5,000 | 9.8折 | 1.2倍 |
| 金卡 | ¥5,000 - ¥20,000 | 9.5折 | 1.5倍 |
| 铂金 | ¥20,000 - ¥50,000 | 9折 | 2倍 |
| 黑金 | > ¥50,000 | 8.5折 | 3倍 |

#### 3.4.3 积分规则

```
获得积分 = 消费金额 × 积分倍率
积分抵扣 = 100积分 = ¥1
```

### 3.5 订单管理规则

#### 3.5.1 订单状态流转

```
待支付 → 已支付 → 已确认 → 已配货 → 已发货 → 已送达 → 已完成
         ↓
       已取消
```

#### 3.5.2 订单处理时效

| 订单类型 | 确认时限 | 配送时限 |
|----------|----------|----------|
| 当日达 | 30分钟内 | 2小时内 |
| 次日达 | 2小时内 | 当日发出 |
| 标准达 | 24小时内 | 72小时内 |

#### 3.5.3 退货退款规则

| 退货场景 | 时限 | 退款方式 |
|----------|------|----------|
| 商品质量问题 | 30天内 | 原支付方式 |
| 非质量问题 | 7天内(食品3天) | 原支付方式/积分 |
| 物流损坏 | 签收24小时内 | 原支付方式 |

### 3.6 门店运营规则

#### 3.6.1 营业时间规则

| 业态 | 常规营业时间 | 特殊时段 |
|------|--------------|----------|
| 便利店 | 24小时/7天 | - |
| 超市 | 8:00-22:00 | 节假日提前/延后 |
| 百货 | 9:30-21:30 | 促销期延时 |

#### 3.6.2 人员配置规则

```
店员数量 = 经营面积 / 人均面积标准 × 时段系数
```

**人均面积标准**:
- 便利店: 30㎡/人
- 超市: 50㎡/人
- 百货: 80㎡/人

### 3.7 供应商管理规则

#### 3.7.1 供应商等级规则

| 等级 | 考核得分 | 合作策略 |
|------|----------|----------|
| A级 | ≥90分 | 战略合作、优先供货 |
| B级 | 75-89分 | 正常合作 |
| C级 | 60-74分 | 观察改进 |
| D级 | <60分 | 淘汰替换 |

#### 3.7.2 账期规则

| 供应商等级 | 账期 | 结算方式 |
|------------|------|----------|
| A级 | 60天 | 月结 |
| B级 | 45天 | 月结 |
| C级 | 30天 | 预付款/现结 |

## 五、应用场景

### 5.1 智能问答

基于本体知识回答零售业务常见问题：
- 商品管理: 商品上架流程、定价原则
- 库存管理: 补货规则、盘点流程
- 客户管理: 会员权益、积分规则
- 促销管理: 活动类型、审批流程

### 5.2 流程自动化

根据业务规则自动触发：
- 库存预警自动补货提醒
- 促销审批流程自动流转
- 会员等级自动调整
- 订单状态自动更新

### 5.3 数据分析

基于规律进行业务洞察：
- 销售数据分析 (销售额、客单价、转化率)
- 库存分析 (周转率、积压预警)
- 客户分析 (RFM分群、CLV计算)
- 促销效果分析 (ROI计算)

### 5.4 知识推理

通过本体关系发现业务洞察：
- 基于商品关联推荐组合促销
- 识别高价值客户制定针对性营销
- 发现滞销商品建议促销清仓
- 预测需求优化补货策略

| 指标类别 | 指标名称 | 计算公式 | 参考值 |
|----------|----------|----------|--------|

| 促销指标 | 促销贡献率 | 促销销售额/总销售额 | <40% |
