## 1. 常见问题

### 1.1 产品基础

#### Q1: 本体论推理引擎支持哪些应用场景？
A: 本产品主要支持以下三大场景：
- **燃气设备选型**: 包括燃气调压箱、计量设备、管道附件等选型推荐
- **采购合规判断**: 根据采购金额和品类属性，判断招标要求、供应商数量等
- **安全生产检查**: 生成检查清单、风险评估、安全建议

#### Q2: 如何接入本体论推理引擎？
A: 接入步骤如下：
1. 确认本体库版本（需V1.0及以上）
2. 准备输入参数（参考测试用例文档）
3. 调用推理接口获取结果
4. 解析JSON响应

#### Q3: 推理结果的置信度如何理解？
A: 置信度分为四个等级：
- **A**: 高置信度，推理依据充分，结果可靠
- **B**: 中置信度，推理依据基本充分，建议复核
- **C**: 低置信度，推理依据不足，需人工确认
- **D**: 无法判断，需要补充更多输入参数

---

### 1.2 功能使用

#### Q4: 输入参数不完整时还能推理吗？
A: 可以，但推理结果置信度会降低。建议提供完整参数以获得更准确的结果。必填参数因场景而异，具体请参考各场景的输入规范。

#### Q5: 推理结果可以导出为报告吗？
A: 当前版本支持JSON格式输出。可通过后处理模块将结果转换为Word/PDF格式报告。

#### Q6: 如何查看推理过程的详细信息？
A: 推理结果中包含`reasoning_chain`字段，记录了完整的推理步骤和依据。

---

### 1.3 业务规则

#### Q7: 燃气调压箱选型的核心逻辑是什么？
A: 核心逻辑如下：
1. 根据用户类型、户数、建筑类型计算用气规模
2. 结合供暖需求修正流量需求
3. 匹配供气压力等级
4. 从本体库中筛选符合条件的型号
5. 输出推荐型号及置信度

#### Q8: 采购金额达到多少需要公开招标？
A: 根据《招标投标法》规定：
- 金额 ≥ 50万元：必须公开招标
- 金额 ≥ 200万元：必须公开招标且需资格预审
- 特种设备采购：至少需要3家供应商参与

#### Q9: 安全生产检查的频率如何确定？
A: 检查频率基于以下因素：
- 场站规模：大型场站每月1次，中型每季度1次，小型每半年1次
- 风险等级：高风险场所增加频次
- 历史问题：存在问题需跟踪整改
- 季节因素：冬季重点防冻检查，夏季重点防暑检查

## 2. 故障排查

### 2.1 推理异常

#### 故障1: 推理返回空结果

**现象**: 调用推理接口后返回空结果或null

**可能原因**:
1. 输入参数格式错误
2. 参数值超出本体库覆盖范围
3. 本体库未正确加载

**排查步骤**:
1. 检查输入JSON格式是否正确
2. 验证参数值是否在有效范围内
3. 检查本体库加载日志
4. 确认接口版本与本体库版本匹配

**解决方法**:
- 修正输入参数格式
- 使用有效范围内的参数值
- 重启服务重新加载本体库

---

#### 故障2: 推理结果置信度为D

**现象**: 推理成功但置信度为D，无法使用

**可能原因**:
1. 输入参数不完整
2. 输入参数组合在本体库中无匹配规则
3. 本体库规则覆盖不全

**排查步骤**:
1. 对比测试用例检查必填参数是否完整
2. 尝试简化输入参数组合
3. 检查本体库是否有对应规则

**解决方法**:
- 补充完整输入参数
- 调整参数值使其匹配现有规则
- 反馈给开发团队补充本体规则

---

#### 故障3: 推理结果不符合预期

**现象**: 推理结果与业务判断不一致

**可能原因**:
1. 本体库规则与实际业务规则有差异
2. 推理逻辑存在缺陷
3. 输入参数理解有误

**排查步骤**:
1. 查看reasoning_chain分析推理过程
2. 对比测试用例验证输入参数
3. 检查本体库对应规则

**解决方法**:
- 修正输入参数
- 反馈问题给本体库维护团队
- 人工复核结果

---

### 2.2 系统异常

#### 故障4: 服务启动失败

**现象**: 服务无法启动，日志显示加载失败

**可能原因**:
1. 本体库文件损坏或缺失
2. 端口被占用
3. 内存不足

**排查步骤**:
1. 检查本体库文件完整性
2. 查看端口占用情况
3. 检查系统资源

**解决方法**:
- 重新加载或修复本体库文件
- 更换端口或释放占用端口
- 增加系统内存

---

#### 故障5: 接口响应超时

**现象**: 推理接口调用超时

**可能原因**:
1. 本体库过大导致加载慢
2. 网络延迟
3. 服务资源不足

**排查步骤**:
1. 检查本体库大小
2. 测试网络延迟
3. 检查服务资源使用情况

**解决方法**:
- 优化本体库结构
- 使用本地部署
- 增加服务资源

---

### 2.3 数据异常

#### 故障6: 推理结果数据异常

**现象**: 返回的数据字段缺失或格式错误

**可能原因**:
1. 本体库数据不完整
2. 输出模板配置错误
3. 版本兼容性问题

**排查步骤**:
1. 核对输出字段定义
2. 检查本体库数据完整性
3. 确认版本兼容性

**解决方法**:
- 补充本体库数据
- 修正输出模板配置
- 升级/降级版本

## 3. 技术FAQ

### 3.1 接口规范

#### Q10: 推理接口的请求格式是什么？
A: 请求格式为JSON，示例：
```json
{
  "scenario": "gas_equipment_selection",
  "parameters": {
    "user_type": "居民",
    "household_count": 100,
    "building_type": "高层建筑",
    "heating_required": true,
    "supply_pressure": "中压B级",
    "usage_area": "住宅区"
  }
}
```

#### Q11: 推理接口的响应格式是什么？
A: 响应格式为JSON，包含以下字段：
```json
{
  "success": true,
  "result": {
    "recommended_model": "RTZ-150/25",
    "capacity": "150m³/h",
    "confidence": "A",
    "reasoning_chain": [...]
  },
  "error": null
}
```

#### Q12: 如何处理接口错误？
A: 错误响应格式如下：
```json
{
  "success": false,
  "result": null,
  "error": {
    "code": "INVALID_PARAMETER",
    "message": "参数household_count值无效"
  }
}
```

### 3.2 本体库

#### Q13: 本体库的结构是怎样的？
A: 本体库采用层级结构：
- **概念层**: 定义实体类型（如设备、供应商、检查项）
- **属性层**: 定义实体属性（如型号、压力、容量）
- **规则层**: 定义推理规则（如选型规则、合规规则）
- **实例层**: 具体实例数据（如RTZ-150/25型号参数）

#### Q14: 如何更新本体库？
A: 更新步骤：
1. 备份当前本体库
2. 导入新版本本体库文件
3. 验证加载成功
4. 运行测试用例验证功能

#### Q15: 本体库版本如何管理？
A: 采用语义化版本号：
- 主版本号：重大架构变更
- 次版本号：新增功能或规则
- 修订号：bug修复或小幅优化

### 3.3 集成部署

#### Q16: 支持哪些部署方式？
A: 支持以下部署方式：
- **本地部署**: 私有化部署到客户机房
- **云端部署**: 部署到阿里云/腾讯云
- **混合部署**: 核心数据本地，业务逻辑云端

#### Q17: 部署的硬件要求是什么？
A: 最低配置要求：
- CPU: 4核
- 内存: 8GB
- 磁盘: 50GB SSD
- 网络: 100Mbps

推荐配置：
- CPU: 8核
- 内存: 16GB
- 磁盘: 100GB SSD
- 网络: 1Gbps

#### Q18: 如何进行性能优化？
A: 性能优化建议：
1. 本体库预加载到内存
2. 使用缓存加速频繁查询
3. 批量处理减少接口调用
4. 异步处理非实时请求

### 3.4 安全保障

#### Q19: 如何保证数据安全？
A: 安全措施：
- 传输层：HTTPS加密
- 存储层：数据加密存储
- 访问层：API密钥认证
- 审计层：完整操作日志

#### Q20: 是否有数据备份机制？
A: 是的，支持：
- 自动备份：每日自动备份
- 手动备份：支持按需备份
- 异地备份：支持备份到异地存储

### 3.5 故障诊断

#### Q21: 如何获取诊断日志？
A: 日志获取方式：
- 控制台日志：实时输出到stdout
- 文件日志：保存在`/var/log/ontology/`目录
- 接口日志：通过API获取

日志级别：DEBUG、INFO、WARN、ERROR

#### Q22: 如何进行问题排查？
A: 排查流程：
1. 查看错误码和错误信息
2. 检查日志详情
3. 核对输入参数
4. 对比测试用例
5. 如仍无法解决，联系技术支持
